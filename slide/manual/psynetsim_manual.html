psynetsimについて

psynetsimは，大水が行っている心理ネットワークのシミュレーションに用いる関数が含まれたパッケージです。適宜関数は追加されます。 以下のコードを実行することで動かすことができます。

remotes::install_github("TakumiOmizu/psynetsim")
library(psynetsim)

psynetsimでは，症状同士が互いに影響し合うネットワークを仮定し，そのネットワークに対して治療的な介入を加えたときに，症状の活性化が時間のなかでどのように変化するかをシミュレーションします。ここでいう「症状」は，抑うつ気分，不安，不眠，疲労感などの心理・行動的な状態を表すノードです。ネットワークのエッジは，ある症状が別の症状をどの程度活性化しやすいかを表します。

このパッケージで扱うシミュレーションは，おおまかには次の考え方に基づいています。

  • 各症状は，それぞれ 0 または 1 の状態をとります。
  • 1 はその症状が活性化している状態，0 は活性化していない状態を表します。
  • ある時点の症状状態は，ネットワークの重み，閾値，直前の症状状態に基づいて確率的に更新されます。
  • 介入は，治療コンポーネントを追加したり，症状の閾値を変化させたりする形で表現されます。
  • 1回のシミュレーションだけでは確率的なゆらぎが大きいため，通常は複数試行を行い，平均的な変化を確認します。

関数

  • 以降ではpsynetsimパッケージに含まれている関数の使い方についてまとめています。 (追記するまでは，Rで？「関数名」を実行して使い方をご確認ください。)

  • 最新はver0.2.0で，閾値に介入する時系列シミュレーション関数(simulate_network_threshold)を追加しました。

現在の主な関数は次の3つです。

関数名 概要
simulate_treatment_network() 症状ネットワークに治療コンポーネントを順に追加し，症状と治療コンポーネントの活性化の変化をシミュレーションする関数です。
simulate_treatment_network_dym() simulate_treatment_network()を拡張し，介入の進行中に症状ネットワーク自体をデータから再推定・更新する関数です。
simulate_network_threshold() 治療コンポーネントを追加するのではなく，症状の閾値に直接介入し，介入開始・終了後の動的変化をシミュレーションする関数です。

共通して必要になる入力

3つの関数では，基本的に以下のような情報を用意します。

W_init

W_initは，初期状態の症状ネットワークを表す正方行列です。行数と列数は症状数に対応します。たとえば6症状を扱う場合は，6行6列の行列を指定します。

行列の各要素は，症状間の結びつきの強さを表します。対角成分は自分自身への影響なので，通常は0にします。

W_init <- matrix(
  c(
    0.0, 0.2, 0.1,
    0.2, 0.0, 0.3,
    0.1, 0.3, 0.0
  ),
  nrow = 3,
  byrow = TRUE
)

b_init

b_initは，各症状の初期閾値を表す数値ベクトルです。長さはW_initの行数，つまり症状数と一致している必要があります。閾値は，症状が活性化するしやすさに関わります。

b_init <- c(0.3, 0.4, 0.2)

target

targetは，どの症状に介入するかを指定する引数です。ただし，関数によって意味が少し異なります。

  • simulate_treatment_network()とsimulate_treatment_network_dym()では，1が介入対象，0が非対象を表します。
  • simulate_network_threshold()では，各症状の閾値にどれだけ介入効果を加えるかを表す数値ベクトルです。0ならその症状には介入しません。
# 1番目と3番目の症状を介入対象にする
target_binary <- c(1, 0, 1)

# 1番目と3番目の症状の閾値に介入する
target_threshold <- c(-0.5, 0, -0.5)

symptom_name

symptom_nameは，症状名を指定する文字ベクトルです。指定しない場合は，関数側でa, b, cやSymptom_1, Symptom_2のような名前が使われます。結果のネットワーク図を読みやすくするため，実際の分析では指定しておくことをおすすめします。

symptom_name <- c("Depressed mood", "Anxiety", "Insomnia")

simulate_treatment_network

simulate_treatment_network()は，症状ネットワークに治療コンポーネント(TC: treatment component)を順番に追加していくシミュレーション関数です。

この関数では，まず介入前のベースライン期間をシミュレーションします。その後，治療コンポーネントを1つずつ追加します。各治療コンポーネントは，targetで指定された症状に対して負の結びつきを持つように導入されます。つまり，治療コンポーネントが活性化すると，対象症状の活性化を下げる方向に働く，というイメージです。最後にフォローアップ期間をシミュレーションします。

実行すると，平均的なネットワーク図と，症状および治療コンポーネントの平均活性化の時系列図が得られます。

simulate_treatment_network(
  W_init,
  b_init,
  target,
  connectivity = 1,
  edge_between_TC = 1,
  weight_bias = 1,
  TB = 1,
  trial = 10,
  baseline_iteration = 10,
  num_TC = 5,
  TC_iteration_per_component = 10,
  follow_up_iteration = 10,
  symptom_name = NULL
)

引数

引数 説明
W_init 初期症状ネットワークを表す正方行列です。行数・列数が症状数になります。
b_init 各症状の初期閾値を表す数値ベクトルです。長さはW_initの行数と一致させます。
target 介入対象の症状を指定します。1が介入対象，0が非対象です。リストまたはベクトルで指定できます。
connectivity ネットワーク全体の結合の強さを調整する係数です。大きいほど，隣接ノードからの影響が強くなります。既定値は1です。
edge_between_TC 治療コンポーネント同士の結びつきの強さを調整する係数です。既定値は1です。
weight_bias 治療コンポーネントから対象症状への負の効果の強さを調整する係数です。大きいほど，対象症状を抑制する方向の結びつきが強くなります。既定値は1です。
TB 治療コンポーネントの閾値を調整する係数です。大きいほど，治療コンポーネントが自発的に活性化しにくくなります。既定値は1です。
trial シミュレーションを何回繰り返すかを指定します。結果は試行間で平均されます。既定値は10です。
baseline_iteration 治療コンポーネントを導入する前のベースライン期間の長さです。既定値は10です。
num_TC 導入する治療コンポーネントの数です。既定値は5です。
TC_iteration_per_component 治療コンポーネントを1つ追加するごとに，何ステップ状態更新を行うかを指定します。既定値は10です。
follow_up_iteration すべての治療コンポーネントを導入した後のフォローアップ期間の長さです。既定値は10です。
symptom_name 症状名を指定する文字ベクトルです。NULLの場合は自動的に名前が付けられます。

返り値

返り値はリストです。

要素 説明
result_plot ネットワーク図と時系列図を並べたプロットです。
result_text 最終時点における症状活性化と治療コンポーネント活性化の平均・標準偏差をまとめた文字列です。

使用例

set.seed(123)

W_init <- matrix(
  c(
    0.0, 0.2, 0.1,
    0.2, 0.0, 0.3,
    0.1, 0.3, 0.0
  ),
  nrow = 3,
  byrow = TRUE
)

b_init <- c(0.3, 0.4, 0.2)
target <- c(1, 0, 1)
symptom_name <- c("Depressed mood", "Anxiety", "Insomnia")

res <- simulate_treatment_network(
  W_init = W_init,
  b_init = b_init,
  target = target,
  trial = 20,
  baseline_iteration = 20,
  num_TC = 3,
  TC_iteration_per_component = 20,
  follow_up_iteration = 20,
  symptom_name = symptom_name
)

res$result_text
res$result_plot

simulate_treatment_network_dym

simulate_treatment_network_dym()は，simulate_treatment_network()を動的ネットワーク版に拡張した関数です。

基本的な流れはsimulate_treatment_network()と同じで，ベースライン期間のあとに治療コンポーネントを順番に追加していきます。大きな違いは，介入の進行中に得られた症状状態の時系列データを使って，症状ネットワークを再推定する点です。再推定には正則化ロジスティック回帰が使われます。これにより，治療が進むなかで症状同士の関係が変化する状況を表現できます。

たとえば，ある症状を抑えることで，別の症状との結びつきが弱くなる，あるいは別の経路が強くなる，といった変化を含めてシミュレーションしたい場合に使います。

simulate_treatment_network_dym(
  W_init,
  b_init,
  target,
  connectivity = 1,
  edge_between_TC = 1,
  weight_bias = 1,
  TB = 1,
  trial = 10,
  baseline_iteration = 10,
  num_TC = 5,
  TC_iteration_per_component = 10,
  follow_up_iteration = 10,
  symptom_name = NULL
)

引数

引数 説明
W_init 初期症状ネットワークを表す正方行列です。
b_init 各症状の初期閾値を表す数値ベクトルです。
target 介入対象の症状を指定します。1が介入対象，0が非対象です。
connectivity ネットワーク全体の結合の強さを調整する係数です。大きいほど，直前の状態が次の状態に与える影響が強くなります。既定値は1です。
edge_between_TC 治療コンポーネント同士の結びつきの強さを調整する係数です。既定値は1です。
weight_bias 治療コンポーネントから介入対象症状への負の結びつきの強さを調整します。既定値は1です。
TB 治療コンポーネントの閾値を調整する係数です。既定値は1です。
trial シミュレーションの繰り返し回数です。既定値は10です。
baseline_iteration 介入前のベースライン期間の長さです。既定値は10です。
num_TC 導入する治療コンポーネントの数です。既定値は5です。
TC_iteration_per_component 治療コンポーネントを追加した後，次の追加・再推定までに何ステップ進めるかを指定します。既定値は10です。
follow_up_iteration 治療コンポーネント導入後のフォローアップ期間です。既定値は10です。
symptom_name 症状名を指定する文字ベクトルです。NULLの場合は，W_initの列名または自動生成された名前が使われます。

返り値

返り値はリストです。

要素 説明
result_plot 症状ネットワーク図と，症状・治療コンポーネントの平均活性化の時系列図を並べたプロットです。
result_text 最終時点における症状活性化と治療コンポーネント活性化の平均・標準偏差をまとめた文字列です。

simulate_treatment_network()との違い

simulate_treatment_network()では，最初に与えた症状ネットワークを基本として，そこに治療コンポーネントを追加していきます。一方，simulate_treatment_network_dym()では，介入中の症状データに基づいて症状ネットワーク部分が更新されます。そのため，介入によって症状間の関係そのものが変化すると考えたい場合には，simulate_treatment_network_dym()のほうが適しています。

ただし，動的にネットワークを再推定するため，TC_iteration_per_componentが極端に短いと，再推定に使えるデータが少なくなります。安定した推定を行いたい場合は，試行数や各フェーズのステップ数をある程度大きく設定することが重要です。

使用例

set.seed(123)

res_dym <- simulate_treatment_network_dym(
  W_init = W_init,
  b_init = b_init,
  target = target,
  trial = 20,
  baseline_iteration = 20,
  num_TC = 3,
  TC_iteration_per_component = 30,
  follow_up_iteration = 20,
  symptom_name = symptom_name
)

res_dym$result_text
res_dym$result_plot

simulate_network_threshold

simulate_network_threshold()は，ver0.2.0で追加された，閾値への介入を扱う時系列シミュレーション関数です。

simulate_treatment_network()とsimulate_treatment_network_dym()では，治療コンポーネントという新しいノードをネットワークに追加します。一方，simulate_network_threshold()では，新しいノードは追加せず，既存の症状ノードの閾値に直接介入します。したがって，「治療によって特定の症状が活性化しにくくなる」「一定期間だけ特定症状の閾値が変わる」といった状況を表現するのに向いています。

この関数では，ベースライン期間のあとにtargetで指定した値をb_initに加えます。その後，指定された期間だけ介入を続け，介入終了時点でその効果を取り除きます。また，介入後の時系列データを用いて，症状ネットワークを複数回再推定します。

simulate_network_threshold(
  W_init,
  b_init,
  target,
  connectivity = 1,
  trial = 10,
  baseline_iteration = 10,
  intervention_duration = 20,
  num_updates = 5,
  update_interval = 10,
  follow_up_iteration = 10,
  symptom_name = NULL
)

引数

引数 説明
W_init 初期症状ネットワークを表す正方行列です。行数と列数は一致している必要があります。
b_init 各症状の初期閾値を表す数値ベクトルです。長さはW_initの行数と一致している必要があります。
target 各症状の閾値に加える介入量を表す数値ベクトルです。長さは症状数と一致している必要があります。0を指定した症状には介入しません。
connectivity ネットワーク全体の結合の強さを調整する係数です。既定値は1です。
trial シミュレーションの繰り返し回数です。既定値は10です。
baseline_iteration 閾値介入を始める前のベースライン期間の長さです。既定値は10です。
intervention_duration 閾値介入を何ステップ継続するかを指定します。既定値は20です。
num_updates 介入開始後にネットワークを何回再推定するかを指定します。既定値は5です。
update_interval ネットワーク再推定の間隔です。たとえば10なら，10ステップごとに症状データを使ってネットワークを再推定します。既定値は10です。
follow_up_iteration 動的更新フェーズの後に追加で実行するフォローアップ期間です。既定値は10です。
symptom_name 症状名を指定する文字ベクトルです。NULLの場合は，W_initの列名または自動生成された名前が使われます。

返り値

返り値はリストです。ほかの2関数よりも，後から結果を詳しく確認しやすい形になっています。

要素 説明
plot_data 各時点・各症状の平均活性化と標準偏差をまとめたデータフレームです。
raw_data 全試行の生データです。次元はおおむね「試行数 × 時点数 × 症状数」です。
final_network 全試行を平均した最終ネットワークです。Wに重み行列，bに閾値ベクトルが含まれます。
network_history_full 各試行におけるネットワークと閾値の更新履歴です。介入開始，介入終了，各更新時点の状態を確認できます。
result_plot 最終試行のネットワーク図と，全体的な活性化の時系列図を並べたプロットです。
result_text 最終時点の全体的な活性化をまとめた文字列です。

targetの指定に注意

simulate_network_threshold()のtargetは，介入対象を1/0で指定するのではなく，閾値に加える量を直接指定します。

たとえば，次のように指定すると，1番目と3番目の症状の閾値に-0.5が加えられます。

target <- c(-0.5, 0, -0.5)

閾値への介入量の符号や大きさは，どのような介入を表現したいかに応じて決める必要があります。実装上は，ベースライン終了後にb <- b + targetとして介入が加えられ，介入終了時点でb <- b - targetとして取り除かれます。

使用例

set.seed(123)

target_threshold <- c(-0.5, 0, -0.5)

res_threshold <- simulate_network_threshold(
  W_init = W_init,
  b_init = b_init,
  target = target_threshold,
  trial = 20,
  baseline_iteration = 20,
  intervention_duration = 30,
  num_updates = 5,
  update_interval = 10,
  follow_up_iteration = 20,
  symptom_name = symptom_name
)

res_threshold$result_text
res_threshold$result_plot

# 各症状の時系列データを確認する
head(res_threshold$plot_data)

# 平均した最終ネットワークを確認する
res_threshold$final_network$W
res_threshold$final_network$b

使い分け

どの関数を使うかは，介入をどのように表現したいかによって変わります。

目的 使う関数
治療コンポーネントを症状ネットワークに追加し，その効果を見たい simulate_treatment_network()
治療中に症状間の関係も変化すると考えたい simulate_treatment_network_dym()
新しい治療ノードではなく，既存症状の閾値を直接変化させたい simulate_network_threshold()

まず基本的な治療コンポーネントの効果を見たい場合は，simulate_treatment_network()から試すとわかりやすいです。介入によってネットワーク構造そのものが変わることを想定する場合は，simulate_treatment_network_dym()を使います。特定症状の活性化しやすさを直接変える介入を表現したい場合は，simulate_network_threshold()を使います。

実行時の注意

  • W_initは正方行列にしてください。
  • b_init，target，symptom_nameの長さは，基本的に症状数と一致させてください。
  • trialを大きくすると結果は安定しやすくなりますが，計算時間は長くなります。
  • baseline_iteration，TC_iteration_per_component，update_intervalなどのステップ数が短すぎると，時系列の変化やネットワーク再推定が不安定になりやすくなります。
  • 乱数を使うため，再現性を確保したい場合は，実行前にset.seed()を指定してください。
